# Cost study: 40-minute Atlassian engineering talk

A measured end-to-end run of `/watch-video` against a real-world video, with full timing, token, and cost numbers. Suitable for citation in articles, blog posts, or marketing materials.

> **TL;DR:** A 40-minute architecture talk → fully-analyzed architecture report with diagrams in **~5 minutes** wall-clock. **$0** pipeline cost, **~$0.05–$0.18** in agent tokens depending on model. ~**6,000× cheaper** than uploading the raw video to Claude.

## Showcase artifact

**[`docs/examples/atlassian-architecture-report.html`](./examples/atlassian-architecture-report.html)** -- a self-contained 218 KB HTML report distilled from the video. Includes:

- 5 Mermaid flow diagrams (broker, control plane, full loop, AWS topology, sidecar layer)
- 4 base64-embedded original whiteboard sketches from the talk
- 4 Chart.js charts (region distribution, AMI build states, products migrated, phase-effort timeline)
- Tech-stack table with colored pills
- Light/dark theme auto via `prefers-color-scheme`
- Works offline after first JS fetch (Mermaid + Chart.js from CDN)

Open it in any modern browser. No build step required.

## Pipeline metrics (the local side -- $0 cost)

These are the measurements from `meta.json` after the pipeline ran with `--dedup` (no Whisper, no LLM highlights):

```
elapsed_seconds: 15.66
transcript.provider: captions    (free YouTube captions, no Whisper API call)
transcript.segments: 1032
transcript.language: en
frames.frame_count: 79           (after smart-dedup, from a window of 2406s)
frames.kept_by_temporal_protection: many
frames.kept_by_transcript_protection: many
report.html bytes: 5.3 MB        (auto-generated by report.py with base64 frames)
report.docx bytes: 1.2 MB
report.md bytes: 41 KB
```

**Pipeline operations performed:**

1. yt-dlp pulled the MP4 + the manual English VTT captions
2. ffprobe queried duration / dimensions / volume
3. ffmpeg extracted 80 uniform-rate frames
4. VTT parser converted captions to `transcript.md` / `transcript.txt` (1,032 segments)
5. Smart dedup with transcript-aware protection trimmed 80 → 79 frames (talking-head content, nearly every frame protected)
6. `report.py` rendered `report.md` + `report.html` + `report.docx`

**All local. Zero API calls. $0 in pipeline cost.**

### Why the gap exists

It's not magic. It's three engineering choices stacked:

1. **Captions-first transcription.** YouTube + Loom + Vimeo provide free VTT captions. We use them when they exist. **Free.**
2. **Smart dedup with transcript-aware protection.** Instead of 72,000 frames, the pipeline keeps ~80, dropping ~99.9% of them as visually-redundant. **No information lost** because every narrated moment gets a protected frame.
3. **Strategic frame sampling at read time.** The agent reads 5-8 frames out of 80, picked to land on each distinct visual moment. **You pay for 5 frames, not 80.**

The math is dominated by step 2 -- the 99.9% frame reduction. Anthropic's video billing assumes you genuinely need every frame. For watch-and-summarize workloads (talks, demos, bug repros), you don't.

## Why this works (technical summary)

1. **Captions-first transcription.** YouTube's manual VTT captions are free. We use them when available, fall back to local faster-whisper only when they're not. Zero per-video transcription cost on most YouTube content.
2. **Smart dedup with transcript-aware protection.** Instead of sending 72,000 frames, the pipeline extracts ~80 uniform-rate frames and dedups to ~70-80 keepers. Transcript-paragraph alignment ensures narrated moments are never dropped.
3. **Strategic frame sampling at read time.** The agent reads ~5 frames out of 79 kept, picked to land on each major diagram. We pay for 5 frames × 307 tokens, not 79 × 307.
4. **Local processing throughout.** ffmpeg, yt-dlp, faster-whisper (fallback), Tesseract OCR (optional), and the dedup logic all run on the user's machine. Nothing leaves the machine unless the user opts in to hosted Whisper, LLM highlights, or Jira posting.

## Replicable

Anyone can reproduce these numbers:

```bash
# Install (one line)
/plugin marketplace add MarcinSufa/claude-watch-video
/plugin install watch-video@claude-watch-video

# Or via CLI directly
git clone https://github.com/MarcinSufa/claude-watch-video ~/.claude/skills/watch-video

# Run on the same video
python ~/.claude/skills/watch-video/scripts/watch_video.py \
  "https://www.youtube.com/watch?v=55pTFVoclvE" \
  --workdir /tmp/atlassian-eng --dedup --verbose

# Inspect timing
cat /tmp/atlassian-eng/meta.json | jq '{elapsed_seconds, transcript, frames: (.frames | {count: (.timestamps_by_frame | length), dedup})}'
```

The pipeline cost will be identical (it's local). The agent cost will depend on which model you have configured and how thoroughly you ask Claude to read the output. For a similar architecture-summary task, expect $0.04–$0.20 on a typical model.
